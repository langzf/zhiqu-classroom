# zhiqu-classroom 初始版文档（V0.3）

## 1. 项目概述

### 1.1 背景
面向K12/职教等学习场景，学生在课后学习中常见痛点包括：
- 课后复习缺乏清晰路径，不知道先学什么
- 知识点理解不牢，错题反复出现
- 学习内容形式单一，注意力与持续性不足
- 家长难以判断孩子真实掌握情况

### 1.2 愿景
打造一个名为“zhiqu-classroom”的学生课后辅导AI应用：
- 录入教材后，自动抽取知识点与能力点
- 基于知识点生成适合课后学习的多模态内容（AI小游戏、短视频讲解、练习题）
- 分阶段建设：先完成“课程与知识点内容引擎”，再升级为“个性化学习闭环”

### 1.3 目标用户
- 学生（核心，重点覆盖课后辅导）
- 家长（查看学习进度、薄弱点和改进建议）
- 教师/教研组（可选，首期主要用于内容校验）

---

## 2. 核心价值

### 2.1 对学生
- 快速获得基于教材知识点的课后学习内容
- 通过小游戏和短视频降低理解门槛
- 在初始版先提升“看得懂、愿意学”，进阶版再实现“学得准、学得快”

### 2.2 对家长
- 看到孩子学习历史总结与阶段变化
- 直观看到薄弱知识点、错误类型和可执行陪学建议

### 2.3 对教师/学校
- 更快发现班级共性薄弱点
- 形成可复用的课后辅导数字资源

## 3. 产品能力设计（初始版）

### 3.1 教材录入与知识结构化
输入方式：
- PDF/Word/PPT上传
- 图片OCR识别
- 手动粘贴文本

处理流程：
1. 文本提取与清洗
2. 章节结构识别
3. 概念/公式/例题抽取
4. 生成知识图谱雏形（章节->知识点->子知识点）

产出：
- 章节大纲
- 知识点列表（定义、难度、先修关系、典型误区）

### 3.2 知识点检索与学习资源生成
- 自动为每个知识点生成：定义、关键术语、常见误区、典型题型
- 建立向量索引，支持语义检索（按章节、学段、难度筛选）
- 支持按“学生当前掌握度”动态推荐资源

### 3.3 AI小游戏生成（MVP）
优先支持2类轻量互动：
- 选择闯关：根据知识点生成题目+即时反馈
- 拖拽配对：术语-定义、公式-场景匹配

生成内容：
- 游戏规则
- 关卡题库
- 正误反馈语
- 难度分层（基础/进阶）

### 3.4 AI短视频学习内容（MVP）
先做“短视频讲解脚本 + 分镜建议”：
- 自动生成30-90秒知识点讲解脚本
- 生成分镜建议（画面元素、字幕要点、节奏）
- 生成素材提示词（便于接第三方视频工具）

### 3.5 学习历史总结
（进阶能力，后期上线）
- 自动记录学习行为：学习时长、练习次数、正确率、错题分布
- 按日/周/月生成学习总结
- 提供阶段趋势：掌握上升、停滞、回落的知识点列表

### 3.6 个性化分析
（进阶能力，后期上线）
- 构建学生画像：知识掌握度、学习节奏、题型偏好、常见错误模式
- 给出知识点级诊断：
  - 概念不清
  - 审题偏差
  - 计算失误
  - 迁移能力不足
- 输出“原因-证据-建议”结构化分析

### 3.7 针对性辅助
（进阶能力，后期上线）
- 生成个性化学习计划（如7天强化计划）
- 自动推送“先修补弱 -> 进阶提升”的练习路径
- 薄弱点触发专项辅导包：
  - 1个短讲解
  - 1个小游戏
  - 5-10道分层练习
- 出题与复习节奏结合记忆曲线与学习曲线动态调整：
  - 在遗忘高风险窗口优先安排复习题
  - 在能力上升窗口提升题目难度与迁移训练
  - 在挫败风险窗口降低难度并增加提示引导

### 3.8 智能学习规划与课程表
（进阶能力，后期上线）
- 结合学生可用时间、学科优先级、薄弱知识点、考试目标自动排课
- 输出日/周课程表，明确每节学习任务（知识点、时长、资源类型、目标）
- 支持跨学科平衡策略：
  - 先补弱后提升
  - 文理交替避免疲劳
  - 控制单次学习时长与难度跃迁
- 动态重排：根据完成情况与测验结果自动调整后续计划

## 5. 系统架构（初版）

### 5.1 架构分层
- 客户端：学生端App/H5 + 家长端小程序
- 应用服务层：账号与学习档案、内容管理、任务编排
- AI能力层：
  - 文档解析（OCR+结构化抽取）
  - 知识点抽取（LLM）
  - 内容生成（游戏/脚本/练习）
  - 学情诊断与推荐引擎（后期）
  - 学习规划与排课引擎（后期）
  - 向量检索（RAG）
- 数据层：
  - 关系库（用户、学习记录、资源元数据）
  - 对象存储（教材、图片、导出文件）
  - 向量库（知识片段索引）

### 5.2 关键流程
1. 录入教材 -> 异步解析与知识点构建
2. 基于知识点生成AI小游戏、短视频脚本和练习任务
3. 学生进行课后学习与练习
4. 系统统计内容使用与完成效果，迭代内容质量
5. （后期）升级为学习闭环：学习记录 -> 诊断 -> 周课程表 -> 练习/复习 -> 复盘

## 6. AI与数据策略

### 6.1 Prompt与模板化
- 为不同学段/学科配置模板Prompt
- 输出结构化JSON，便于渲染与二次分析

### 6.2 个性化推荐策略
（后期能力）
- 输入：学习历史、错题数据、知识点掌握度
- 输出：今日学习建议、优先补弱列表、资源组合包
- 约束：控制难度跃迁，避免推荐跨度过大

### 6.3 智能排课策略
（后期能力）
- 输入：学生可学习时段、科目目标、知识点优先级、考试时间
- 输出：日/周课程表（任务项、时长、资源、验收标准）
- 规则：
  - 每日优先安排1-2个薄弱知识点
  - 同一学科连续学习时长不超过阈值
  - 每周预留复盘时段（错题回顾+阶段测验）
- 调整机制：未完成任务自动顺延并降级粒度，避免计划堆积

### 6.4 记忆曲线与学习曲线驱动策略
（后期能力）
- 目标：在“即将遗忘”前安排最小必要复习，在“可提升”阶段安排最有效进阶训练
- 记忆曲线建模（按知识点）：
  - 维护记忆强度 `memory_strength` 与遗忘风险 `forgetting_risk`
  - 结合最近学习时间、复习次数、答题正确率、反应时动态估计下次复习时间
  - 复习触发规则：当 `forgetting_risk` 超阈值时，自动插入短时复习任务
- 学习曲线建模（按知识点与题型）：
  - 维护能力增长斜率 `learning_velocity` 与稳定度 `stability_score`
  - 若连续正确且反应时间下降，则提升难度与迁移题比例
  - 若连续错误或耗时异常，则回退到基础题并补充讲解
- 题目生成策略（核心）：
  - 题目组成建议：`60%` 巩固题 + `30%` 进阶题 + `10%` 挑战题（可按学生状态动态调整）
  - 每次任务混入少量“间隔复习题”，覆盖近期高遗忘风险知识点
  - 错题重练采用“变式重做”而非原题重复，避免机械记忆
- 个体化自适应：
  - 高专注短时型：缩短单任务时长，提高任务切换频率
  - 低稳定长时型：降低初始难度，增加正反馈密度
  - 考前冲刺型：提高重点考点覆盖与回顾频次

### 6.5 质量控制
- 规则校验：术语一致性、答案唯一性、难度标注合理性
- 模型校验：对生成资源进行自动评分
- 人工抽检：关键学科与年级定期抽检

### 6.6 安全与合规
- 教材版权与上传授权提示
- 未成年人数据脱敏与最小化采集
- 生成内容安全过滤与审核机制

## 7. 指标体系（首期）

### 7.1 产品指标
- 教材解析成功率
- 知识点抽取准确率（抽检）
- 学生端内容使用率（小游戏/短视频/练习）

### 7.2 学习效果指标
- 周活跃学习天数
- 课后任务完成率
- 知识点内容学习完成率（游戏/视频/练习）
- 知识点测验通过率（首测与复测对比）

### 7.3 运营指标
- 周活学生数
- 人均学习时长
- 次月留存率
- 家长周报查看率

## 8. 里程碑计划（示例）

### Phase 0（2周）
- 完成需求梳理、原型图、技术选型
- 明确首批学科与教材范围

### Phase 1（4-6周）
- 完成教材解析+知识点抽取
- 上线知识点内容生成（AI游戏+视频脚本+练习）

### Phase 2（4周）
- 上线课后辅导学习流程（学生学习入口+任务完成反馈）
- 优化内容质量与知识点覆盖率

### Phase 3（4周）
- 进阶能力启动：学习历史总结与学生画像
- 启动个性化诊断、薄弱点强化与智能课程表
- 验证学习闭环效果并迭代推荐策略

## 9. 风险与应对
- 风险：教材格式复杂导致解析不稳定  
  应对：优先支持高频格式，建立失败重试与人工修正工具

- 风险：AI生成内容质量波动  
  应对：模板约束+规则校验+抽检机制

- 风险：学生短期新鲜感后活跃下降  
  应对：引入阶段目标、连续学习奖励、节奏化任务设计

- 风险：个性化推荐不准确  
  应对：先从规则+模型混合策略起步，持续用学习反馈迭代

- 风险：课程表过于理想化，学生难以坚持  
  应对：引入弹性任务槽、最小可完成单元和自动重排机制

- 风险：首期范围过大导致交付延期  
  应对：首期严格聚焦“课程录入+知识点梳理+内容生成+课后学习”，进阶能力分期上线

---

## 10. 下一步建议
1. 先选定一个学科（建议数学）和一个年级做MVP验证
2. 首期重点打通“课程录入 -> 知识点梳理 -> AI游戏/视频/练习生成 -> 课后学习”
3. 进阶阶段再上线“学习记录 -> 诊断 -> 周课程表 -> 练习/复习 -> 复盘”
4. 用50-100名学生做4周试点，先验证内容有效性，再验证个性化闭环
